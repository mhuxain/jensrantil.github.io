ZeroMQ

This post is partially related to my previous blog post about Rewind and Gorewind.

My Rewind and Gorewind implementations both uses ZeroMQ. ZeroMQ is a library that enables a programmer to easily send messages between and within applications without having a central broker. By using smart batching ZeroMQ can achieve very high throughput, while still achieving low latency for low frequency messaging.

The above description is the one usually given about ZeroMQ, but there's more to it. ZeroMQ supports a couple of different common messaging patterns; filtering pubsub, request/reply and push/pull. These can be combined in various ways to create really various messaging setups.

The reason why I chose ZeroMQ for Rewind and Gorewind were for a couple of different reasons:

  • I wanted Rewind and Gorewind to be used in polyglot environments, supporting clients written in multiple programming languages, using different platforms. ZeroMQ supports a bunch of platforms.
  • I did not want to deal with messaging code myself.
  • It had a great reference documentation, as well as guide.
  • It has a vibrant community that's open for contributions. I was given commit access to a Github repository within a day from my first pull request!
  • I was confident that messaging was not going to be a bottleneck for a long time!

The ZeroMQ Guide is a great source of inspiration when it comes to learning about how to build distributed applications using messaging (and ZeroMQ). It not only teaches many of the options you have as a developer, but also the fact that doing distributed systems the right way is hard.
